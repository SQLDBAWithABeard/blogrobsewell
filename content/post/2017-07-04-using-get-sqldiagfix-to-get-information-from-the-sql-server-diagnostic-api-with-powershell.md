<P>The SQL Server Diagnostics Preview was <A href="https://blogs.msdn.microsoft.com/sql_server_team/sql-server-diagnostics-preview/" rel=noopener target=_blank>announced just over a week ago</A> It includes an add-on for SQL Server Management Studio to enable you to analyse SQL Server memory dumps and view information on the latest SQL Server cumulative updates for supported versions of SQL Server. <A href="https://twitter.com/blobeater1" rel=noopener target=_blank>Arun Sirpal</A> has written a<A href="https://blobeater.blog/2017/06/27/using-sql-server-diagnostics-preview/" rel=noopener target=_blank> good blog post showing how to install it and use it in SSMS to analyse dumps</A>.</P>
<P>There is also a <A href="https://ecsapi.portal.azure-api.net/" rel=noopener target=_blank>developer API available</A> so I thought I would write some PowerShell to consume it as there are no PowerShell code examples available in the documentation!</P>
<P>In a <A href="https://blog.robsewell.com/creating-a-powershell-module-and-tdd-for-get-sqldiagrecommendations/">previous post</A> I have explained how I created the module and <A href="https://github.com/SQLDBAWithABeard/SQLDiagAPI" rel=noopener target=_blank>a GitHub repository</A> and used Pester to help me to <A href="https://blog.robsewell.com/powershell-module-for-the-sql-server-diagnostics-api-1st-command-get-sqldiagrecommendations/" rel=noopener target=_blank>develop the first command Get-SQLDIagRecommendations</A>. At present the module has 5 commands, all for accessing the Recommendations API.</P>
<P>This post is about the command Get-SQLDiagFix which returns the Product Name, Feature Name/Area, KB Number, Title and URL for the Fixes in the Cumulative Updates returned from the SQL Server Diagnostics Recommendations API.</P>
<H2>PowerShell Gallery</H2>
<P>The module is <A href="https://www.powershellgallery.com/packages/SQLDiagAPI/" rel=noopener target=_blank>available on the PowerShell Gallery</A> which means that you can install it using</P><PRE class="lang:ps decode:true">Install-Module SQLDiagAPI</PRE>
<P>as long as you have the latest version of the PowerShellGet module. This is already installed in Windows 10 and with WMF 5 but you can install it on the following systems</P>
<UL>
<LI>Windows 8.1 Pro 
<LI>Windows 8.1 Enterprise 
<LI>Windows 7 SP1 
<LI>Windows Server 2016 TP5 
<LI>Windows Server 2012 R2 
<LI>Windows Server 2008 R2 SP1 </LI></UL>
<P><A href="https://msdn.microsoft.com/en-us/powershell/gallery/readme" rel=noopener target=_blank>following the instructions here.</A></P>
<P>If you are not running your PowerShell using a local administrator account you will need to run</P><PRE class="lang:ps decode:true">Install-Module SQLDiagAPI -Scope CurrentUser</PRE>
<P>to install the module.</P>
<P>If you can’t use the PowerShell Gallery you can install it <A href="https://github.com/SQLDBAWithABeard/SQLDiagAPI/blob/master/install.md" rel=noopener target=_blank>using the instructions on the repository</A></P>
<H2>API Key</H2>
<P>To use the API you need an API Key. An API Key is a secret token that identifies the application to the API and is used to control access. You can <A href="https://ecsapi.portal.azure-api.net/" rel=noopener target=_blank>follow the instructions here </A>to get one for the SQL Server Diagnostics API.</P>
<P><IMG title="" class="alignnone size-full wp-image-6331" alt="01 - APIKey" src="https://blog.robsewell.com/assets/uploads/2017/06/01-apikey.png?resize=630%2C157&amp;ssl=1" width=630 height=157 data-recalc-dims="1" loading="lazy" data-large-file="https://blog.robsewell.com/assets/uploads/2017/06/01-apikey.png?fit=630%2C157&amp;ssl=1" data-medium-file="https://blog.robsewell.com/assets/uploads/2017/06/01-apikey.png?fit=300%2C75&amp;ssl=1" data-image-description="" data-image-title="01 – APIKey" data-image-meta='{"aperture":"0","credit":"","camera":"","caption":"","created_timestamp":"0","copyright":"","focal_length":"0","iso":"0","shutter_speed":"0","title":"","orientation":"0"}' data-comments-opened="1" data-orig-size="1171,292" data-orig-file="https://blog.robsewell.com/assets/uploads/2017/06/01-apikey.png?fit=1171%2C292&amp;ssl=1" data-permalink="https://blog.robsewell.com/creating-a-powershell-module-and-tdd-for-get-sqldiagrecommendations/01-apikey/#main" data-attachment-id="6331"></P>
<P>You will need to store the key to use it. I recommend saving the API Key using the <A href="https://msdn.microsoft.com/en-us/powershell/reference/5.1/microsoft.powershell.utility/export-clixml" rel=noopener target=_blank>Export-CliXML</A> command as described by Jaap Brasser <A href="http://www.jaapbrasser.com/tag/export-clixml/" rel=noopener target=_blank>here .</A></P><PRE class="lang:ps decode:true">Get-Credential | Export-CliXml -Path "${env:\userprofile}\SQLDiag.Cred"</PRE>
<P>You need to enter a username even though it is not used and then enter the API Key as the password. It is saved in the root of the user profile folder as hopefully, user accounts will have access there in most shops.</P>
<P>This will save you from having to enter the APIKey every time you run the commands as the code is looking for it to be saved in that file.</P>

<P>Once you have installed the module and the APIKey it will be available whenever you start PowerShell. The first time you install you&nbsp; may need to run</P><PRE class="lang:ps decode:true">Import-Module SQLDiagAPI</PRE>
<P>to load it into your session. Once it is loaded you can view the available commands using</P><PRE class="lang:ps decode:true">Get-Command -Module SQLDiagAPI</PRE>

<P>You can find out <A href="https://github.com/SQLDBAWithABeard/SQLDiagAPI" rel=noopener target=_blank>more about the commands on the GitHub Repository&nbsp;</A> and the Help files <A href="https://github.com/SQLDBAWithABeard/SQLDiagAPI/tree/master/docs" rel=noopener target=_blank>are in the documentation.</A></P>

<H2>Get All Of The Fixes</H2>
<P>The easiest thing to do is to get all of the available fixes from the API. This is done using</P><PRE class="lang:ps decode:true">Get-SQLDiagFix</PRE>
<P>which will return all 123 Fixes currently referenced in the API.</P>
<P><IMG class="alignnone size-full wp-image-6741" alt="03 get-sqldiagfix.png" src="https://blog.robsewell.com/assets/uploads/2017/07/03-get-sqldiagfix.png?resize=630%2C293&amp;ssl=1" width=630 height=293 data-recalc-dims="1" loading="lazy" data-large-file="https://blog.robsewell.com/assets/uploads/2017/07/03-get-sqldiagfix.png?fit=630%2C293&amp;ssl=1" data-medium-file="https://blog.robsewell.com/assets/uploads/2017/07/03-get-sqldiagfix.png?fit=300%2C140&amp;ssl=1" data-image-description="" data-image-title="03 get-sqldiagfix" data-image-meta='{"aperture":"0","credit":"","camera":"","caption":"","created_timestamp":"0","copyright":"","focal_length":"0","iso":"0","shutter_speed":"0","title":"","orientation":"0"}' data-comments-opened="1" data-orig-size="2064,960" data-orig-file="https://blog.robsewell.com/assets/uploads/2017/07/03-get-sqldiagfix.png?fit=2064%2C960&amp;ssl=1" data-permalink="https://blog.robsewell.com/using-get-sqldiagfix-to-get-information-from-the-sql-server-diagnostic-api-with-powershell/03-get-sqldiagfix/#main" data-attachment-id="6741"></P>
<P>That is just a lot of information on the screen. If we want to search through that with PowerShell we can use Out-GridView</P><PRE class="lang:ps decode:true">Get-SQLDiagFix | Select Product, Feature, KB, Title | Out-GridView</PRE>

<P>Or maybe if you want to put them in a database you could use <A href="https://dbatools.io" rel=noopener target=_blank>dbatools</A></P>

<H2>Open the KB Article Web-Page</H2>
<P>As well as getting the title and KB number of the fix, you can open the web-page. This code will open the fixes for all SP1 products in the feature area like al in Out-GridView and enable you to choose one (or more) and open them in your default browser</P><PRE class="lang:ps decode:true">Get-SQLDiagProduct SP1 | Get-SQLDiagFix -Feature (Get-SQLDiagFeature -Feature al) `
